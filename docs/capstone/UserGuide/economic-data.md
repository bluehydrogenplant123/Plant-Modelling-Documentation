# Configure Economic Data

Use this page when you need to enter economic values and map them to model variables.

## Before You Start

- Open a saved diagram.
- Add the nodes and ports you want to map economic values to.
- For Multi-TP economics, configure time periods before editing TP-specific economic rows.

## Open an Economic Panel

1. Open the **Economic** section in the header toolbar.
2. Click the economic panel you need, such as a cost, expense, revenue, or objective panel.
3. In the **Economic - `<panel>`** dialog, edit the entity table and the **Mappings** table.

## Add or Edit an Economic Entity

1. In the entity table, click **Add `<panel>`** if you need a new row.
2. Fill or edit **Generated by**, the entity name, **Value**, **Unit**, and **Type**.
3. For cost rows, edit **Uncertainty** when the field is shown.
4. Click **Save**.

## Map an Entity to a Model Variable

1. In **Mappings**, click **Add Mapping**.
2. Select **Network**.
3. Select the economic entity name.
4. Select **Node**.
5. Select **Port**.
6. Select **Var**.
7. Click **Save**.

## Import or Export Economic Rows

1. Open the relevant economic panel.
2. Click **Import** to load `.csv`, `.xlsx`, or `.xls` economic data.
3. Click **Export** to download the current panel data.
4. Review imported rows before saving.

## Configure Multi-TP Economic Rows

1. Open the **Multi-TP** section.
2. Use the **Economic** dropdown.
3. Select the panel you need.
4. Choose an entity name from the panel selector.
5. Edit **From TP** and **To TP** ranges where shown.
6. Use **Add TP Row** or **Add Mapping Row** for additional TP-specific rows.
7. Click **Save**.

## Result

Economic entities and mappings are saved for the diagram. Multi-TP rows can be scoped to TP ranges when the workflow is opened from the **Multi-TP** section.

## Troubleshooting

| Problem | What To Check |
| --- | --- |
| **Economic** controls are disabled | A computation may be running, or the current mode may be read-only. |
| A node or port is missing from **Mappings** | Confirm the node exists and has loaded model data. |
| Imported rows do not appear as expected | Check workbook or CSV columns, entity names, units, and TP ranges. |
| Multi-TP economic rows are unavailable | Configure and save time periods first. |
