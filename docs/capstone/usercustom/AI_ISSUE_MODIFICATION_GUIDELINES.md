# AI Issue Modification Guidelines

This page was added during the Docusaurus migration because it is referenced by
[`AI_DEVELOPMENT_WORKFLOW`](./AI_DEVELOPMENT_WORKFLOW.md) but was not present in the original
`docs/` directory snapshot.

Use these guardrails when changing issue-scoped work:

- Read the related docs before editing code.
- Limit the change to the files directly needed for the issue.
- Do not fix unrelated problems in the same pass.
- If you discover adjacent issues, document them separately instead of folding them into the current task.
- Verify the affected behavior with the smallest relevant test or smoke check.
